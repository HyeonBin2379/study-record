# study-record

## 개요

* 블로그/노션 공부 기록(2024~) 취합용 리포지토리

## 수행할 작업 목록

* TIL 작성
* 기술 블로그, 노션 등으로 파편화된 이전 공부 기록 정리

## 작업 규칙

### TIL 작성

* 이 작업은 매일 최소 1건씩은 진행하는 것이 원칙
* TIL 작성부터 우선적으로 진행

### 이전 공부 기록 취합

* 이 작업은 점진적으로 진행
* 이전 공부 기록 파일의 저장 경로: `/작성연도/카테고리1[/카테고리2]/제목_날짜.md`
  * 작성연도: 공부 기록을 최초로 작성했던 해를 기준으로 작성 
  * 카테고리1: 당시 무엇을 공부했는지
  * 카테고리2(선택): 당시 공부했던 책/강의 등으로 구분하거나, 직접 파일명을 작성(책/강의의 공부 내용을 정리한 경우, 정리한 내용의 출처를 명시할 것)
  * 파일명 작성 형식: `제목_날짜.md`로 작성하되, 파일명의 날짜는 다음을 기준으로 작성
    * 기술 블로그: 포스팅 당시 날짜를 기준으로 작성
    * 노션: 첫번째 버전을 기준으로 작성
  * 파일 저장 예시: 운영체제 공부 -> `/2026/os/thread_20260101.md`
* 기술 블로그에 정리했던 기록은 작성 당시의 원본 그대로 저장
* 노션에 작성했던 기록은 내용을 보충해야 하는 경우가 존재하여 다음과 같이 작성 
  * 파일명 작성 규칙은 상기한 규칙을 준수
  * 내용 보강이 아닌 단순 오탈자 수정의 경우, 기존 파일명 유지
  * 이미 동일한 내용으로 리포지토리에 저장된 파일이 존재하지만, 추가적인 내용 보강이 필요하다고 판단되는 경우에는 기존 파일에서 해당 작업을 진행
* 작성한 내용에 포함된 이미지 파일의 저장 경로: `/images/작성연도/카테고리1[/카테고리2]/제목_날짜_일련번호.png`
